Lyvo Docs provides a rich set of custom components to make your documentation beautiful and interactive. Thanks to MDX, you can use these components directly within your markdown files without needing to import them (except where noted).

## Callouts

Use callouts to highlight important information, warnings, or tips.

```mdx
<Callout type="default" title="Note">
	This is a standard default callout.
</Callout>
<Callout type="warning" title="Warning">
	Make sure you read this before proceeding.
</Callout>
<Callout type="danger" title="Danger">
	This action is destructive and cannot be undone.
</Callout>
<Callout type="success" title="Success">
	Your installation was completed successfully.
</Callout>
```

## Code Blocks

Standard markdown code blocks are automatically enhanced with syntax highlighting (via Shiki) and a beautiful copy-to-clipboard button.

```tsx showLineNumbers
import { useState } from 'react';

export function Counter() {
	const [count, setCount] = useState(0);
	return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

_(Note: The code block above is enhanced automatically by the pre tag wrapper!)_

## Tabs

Tabs are perfect for showing alternative instructions, such as different package managers or operating systems.

```mdx
<Tabs>
	<TabItem value="pnpm">
		<Code code="pnpm add my-package" lang="bash" />
	</TabItem>
	<TabItem value="npm">
		<Code code="npm install my-package" lang="bash" />
	</TabItem>
	<TabItem value="yarn">
		<Code code="yarn add my-package" lang="bash" />
	</TabItem>
</Tabs>
```

## Steps

The Steps component allows you to create clear, vertical tutorials.

```mdx
<Steps>
	<Step title="Create a new file">Create a new file in your `src/content/docs` directory.</Step>
	<Step title="Add frontmatter">
		Add the required `title` and `description` to the top of the file.
	</Step>
	<Step title="Write some content">
		Start writing your markdown or MDX content right below the frontmatter.
	</Step>
</Steps>
```

## File Trees

Display project directory structures cleanly with interactive, collapsible folders.

```mdx
<FileTree>
	<Folder name="src" defaultOpen>
		<Folder name="components">
			<File name="Button.tsx" />
		</Folder>
		<File name="config.ts" />
	</Folder>
	<File name="package.json" />
</FileTree>
```

## Standard Elements

Lyvo Docs automatically upgrades standard markdown elements too.

### Tables

Tables automatically receive horizontal scrolling on mobile devices to prevent layout breakage.

| Feature         | Lyvo Docs   | Other Frameworks  |
| :-------------- | :---------- | :---------------- |
| Performance     | Outstanding | Great             |
| Customizability | Unlimited   | Limited by config |
| Setup Time      | Seconds     | Minutes           |

### External Links

External links like [GitHub](https://github.com) automatically receive a trailing icon and open in a new tab!

### Images

Images use Astro's native image optimization pipeline automatically. Just use standard markdown syntax: `![Alt Text](@/assets/image.svg)`.

## Accordion

Use accordions to display a list of high-level options that can expand/collapse to reveal more information.

```mdx
<Accordion
	items={[
		{ title: 'Is it accessible?', content: 'Yes. It adheres to the WAI-ARIA design pattern.' },
		{
			title: 'Is it styled?',
			content: 'Yes. It comes with default styles that match the rest of the documentation.'
		},
		{
			title: 'Is it animated?',
			content: "Yes. It's animated by default, but you can disable it if you prefer."
		}
	]}
/>
```

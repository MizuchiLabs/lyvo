Getting up and running with Lyvo Docs is incredibly straightforward. Since you have access to the raw source code, there are no complicated abstractions to fight against.

<Callout type="default" title="Requirements">
	Ensure you have **Node.js 20+** and your preferred package manager installed before you begin.
</Callout>

## Quick Start

<Steps>
  <Step title="Clone the template">
    Start by cloning the repository to your local machine and navigating into the directory:

    ```bash
    git clone https://github.com/mizuchilabs/lyvo.git my-docs
    cd my-docs
    ```

  </Step>

  <Step title="Install dependencies">
    Install the project dependencies using your preferred package manager. We recommend `pnpm` for the fastest experience.

<Tabs>
	<TabItem value="pnpm">
		<Code code="pnpm install" lang="bash" />
	</TabItem>
	<TabItem value="npm">
		<Code code="npm install" lang="bash" />
	</TabItem>
	<TabItem value="yarn">
		<Code code="yarn install" lang="bash" />
	</TabItem>
	<TabItem value="bun">
		<Code code="bun install" lang="bash" />
	</TabItem>
</Tabs>

  </Step>

  <Step title="Start the development server">
    Fire up the local development server to see your documentation live.

<Tabs>
	<TabItem value="pnpm">
		<Code code="pnpm dev" lang="bash" />
	</TabItem>
	<TabItem value="npm">
		<Code code="npm run dev" lang="bash" />
	</TabItem>
	<TabItem value="yarn">
		<Code code="yarn dev" lang="bash" />
	</TabItem>
	<TabItem value="bun">
		<Code code="bun run dev" lang="bash" />
	</TabItem>
</Tabs>

    Your site is now running at [http://localhost:4321](http://localhost:4321). Any changes you make to the `.mdx` files will instantly reflect in your browser!

  </Step>
</Steps>

## Building for Production

When you are ready to deploy your site, build it for production. This generates a highly optimized static version of your site in the `dist` directory, and runs **Pagefind** to generate the offline search index.

```bash
pnpm build
```

Preview the final production build using:

```bash
pnpm preview
```

Lyvo Docs is designed to be easily customized. Since you have full access to the source code, you can tweak anything, but the most common configuration options are consolidated in one easy-to-read file.

## Site Metadata (`src/config.ts`)

The primary configuration file for your site is `src/config.ts`. Here you can configure the global properties, navigation, and sidebar ordering.

```ts title="src/config.ts"
export const SITE = {
	title: 'Lyvo Docs',
	description: 'A modern, minimalistic documentation theme for Astro.',
	repo: {
		url: 'https://github.com/mizuchilabs/lyvo',
		type: 'github' // Supports: github, gitlab, gitea
	},
	openapi: {
		url: '/openapi.json' // Auto-generates an API reference page!
	},
	social: {
		discord: 'https://discord.gg/...',
		x: 'https://x.com/...'
	},
	nav: [{ title: 'Documentation', href: '/docs/introduction' }],
	logo: {
		type: 'both', // "text", "logo", or "both"
		src: 'logo.svg', // Loaded from src/assets/
		width: 28,
		height: 28
	},
	enableEdit: true, // Enables the "Edit Page" button on docs
	categoryOrder: ['introduction', 'overview', 'components', 'changelog'] // Sidebar sorting
};
```

### Sidebar Ordering

The `categoryOrder` array is the easiest way to manage your sidebar. It allows you to freely intermix both single files (like `introduction.mdx`) and folders (like the `overview` folder) to create exactly the navigation flow you want.

Items not listed in the array will automatically fall back to alphabetical sorting at the bottom.

### Repository Links

By providing a `repo.url` and setting the `repo.type` to either `"github"`, `"gitlab"`, or `"gitea"`, Lyvo Docs will automatically:

1. Display the correct icon link in the top navigation bar.
2. Enable the **"Edit Page"** button (if `enableEdit: true`) on every documentation page, pointing directly to the corresponding `.mdx` file in your remote repository!

## Theming & Colors

Lyvo Docs uses standard Shadcn CSS variables for theming, located in `src/styles/global.css`.

You can easily change the primary accent colors, background colors, and border styles by modifying these HSL variables. The theme natively supports both light and dark modes!

```css title="src/styles/global.css"
:root {
	--background: 0 0% 100%;
	--foreground: 0 0% 3.9%;
	/* ... */
}

.dark {
	--background: 0 0% 3.9%;
	--foreground: 0 0% 98%;
	/* ... */
}

@theme inline {
	--color-background: var(--background);
	--color-foreground: var(--foreground);
	/* ... */
}
```

Lyvo Docs utilizes Astro's Content Collections to manage your documentation. This provides type-safe frontmatter, automatic routing, and excellent performance.

## Frontmatter Schema

Every `.mdx` file inside the `src/content/docs` directory requires a frontmatter block at the top of the file. This metadata is used to generate the layout, sidebar, and SEO tags.

```yaml
---
title: My Page Title
description: A short summary of what this page covers.
order: 3
---
```

### Properties

| Property      | Type     | Description                                                                                                                    |
| :------------ | :------- | :----------------------------------------------------------------------------------------------------------------------------- |
| `title`       | `string` | **Required.** The main title of the page, displayed at the top and in the sidebar.                                             |
| `description` | `string` | _Optional._ A brief subtitle displayed below the title and used for the meta `<meta name="description">` tag.                  |
| `order`       | `number` | _Optional._ Determines the sorting order of the pages in the sidebar and Next/Previous navigation. Lower numbers appear first. |

## Routing and Folders

The URL of your documentation page is determined by the file's path.

- `src/content/docs/introduction.mdx` becomes `/docs/introduction`
- `src/content/docs/components/components.mdx` becomes `/docs/components/components`

You can create subfolders inside `src/content/docs` to automatically categorize your pages. The folder name will be used as the category name in the sidebar (e.g., `getting-started` becomes "Getting Started"). Files directly in `src/content/docs` will appear at the root level of the sidebar.

## Auto-Imported Components

Most of the custom components provided by Lyvo Docs (like `<Callout>`, `<Steps>`, `<FileTree>`, `<Tabs>`) are automatically injected into your MDX scope.

This means you do **not** need to write `import { Callout } from '@/components/Callout'` at the top of your markdown files. You can just start using them immediately!

## Table of Contents

The Table of Contents (TOC) on the right side of the page is automatically generated from your markdown headings.

- It captures `##` (H2) and `###` (H3) headings.
- It automatically tracks your scroll position and highlights the active section.
- To exclude a section from the TOC, you can simply use a smaller heading like `####` (H4).
